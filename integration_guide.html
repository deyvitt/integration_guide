🏗️ Architecture: BRAIN as a persistent memory bank

📥 Memory Injection

A small adapter (or hook) reads BRAIN’s memory tensor and adds it to the hidden state of every transformer layer.

hidden = hidden + memory_proj(brain_vector)

🧬 Hebbian Update

After each response, user feedback (👍/👎) triggers mesh.record_outcome(), strengthening/weakening the pathways that produced the memory.

   Input tokens → Embedding + memory → Layer 1 (with memory) → ... → Layer 32 (with memory) → LM head
                           ↑                                                      ↑
                     BRAIN vector                                      BRAIN vector refreshed

Memory is never lost – it is re‑injected at every layer, on every token.

🛠️ Which files to modify (example: Phi‑3‑mini)

We work directly inside the HuggingFace transformers library or a forked model repository.

File: modeling_phi3.py (or equivalent for Llama/Mistral)

class Phi3DecoderLayer(nn.Module):
    def forward(self, hidden_states, ..., memory_tensor=None):
        # original attention
        residual = hidden_states
        hidden_states = self.input_layernorm(hidden_states)
        hidden_states = self.self_attn(hidden_states, ...)
        hidden_states = residual + hidden_states

        # ---- BRAIN INJECTION ----
        if memory_tensor is not None:
            # memory_tensor shape: [batch, 1, hidden_dim] – added to every token
            hidden_states = hidden_states + memory_tensor
        # -------------------------

        residual = hidden_states
        hidden_states = self.post_attention_layernorm(hidden_states)
        hidden_states = self.mlp(hidden_states)
        hidden_states = residual + hidden_states
        return hidden_states

Similarly, modify the main Phi3Model.forward to accept a memory_tensor argument and pass it down through all layers.

🔁 Generating the memory tensor from BRAIN

At each forward pass (every new user message), the LLM calls BRAIN’s mesh to produce a dynamic memory vector.

# In your generation loop (pseudo‑code)
session_id = user_id
user_embedding = embedder.encode(user_message)   # 384‑d
memory_tensor = await mesh.refine(user_embedding, session_id)
# memory_tensor shape: [1, 1, hidden_dim] (e.g., 3072)
output = model.generate(..., memory_tensor=memory_tensor)

The mesh internally updates its VNIs and Hebbian weights based on previous outcomes. Over time, the memory vector becomes more relevant.

❓ Frequently asked questions

Q: Does BRAIN work with frozen models?
A: Yes – via forward hooks (no retraining). For best results, we recommend light fine‑tuning (1‑2 epochs) to teach the model to attend to the memory tensor.

Q: What is the latency overhead?
A: <10ms per forward pass (on GPU). The mesh runs in a separate thread and only produces one vector per user message.

Q: Is the memory persistent across server restarts?
A: Yes – we save the mesh state to disk and reload it.
